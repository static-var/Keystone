# Keystone UI Module

## Intent
Shape user-facing flows, screens, component behavior, information architecture, and interface copy.

## Load when
The user asks for UI/UX structure, screen behavior, product flows, component states, or interaction requirements.

## Allowed mutation
Only UI specs, mock content, or scoped UI files the user asks to change.

## Must not
Make broad visual-brand decisions better handled by `design`, or implement code without a `build` handoff.

## May call
`design` for visual direction; `write` for copy; `breakdown` for implementation planning.

## Handoff
Provide UI decisions, states, edge cases, and next module if build or design work remains.

## Exit gate
Primary user path, states, and constraints are explicit.

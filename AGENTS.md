# Keystone agent instructions

Be concise.

## Skill authoring

When creating, editing, auditing, or validating any `SKILL.md` or skill-supporting reference file in this repo, first read and apply:

```text
.agents/skills/writing-great-skills/SKILL.md
.agents/skills/writing-great-skills/GLOSSARY.md
```

Use those files as the local source of truth for skill-writing quality: predictability, model invocation, context pointers, leading words, information hierarchy, completion criteria, progressive disclosure, single source of truth, duplication, sprawl, no-ops, and negation.

Do not use `maintainers/skill-engineering.md`; Keystone-specific skill-writing guidance belongs here and in the writing-great-skills references.
